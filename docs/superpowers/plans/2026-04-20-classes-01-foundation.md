# Classes — Plan 1: Foundation & Admin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create the `classes/` Django app with full data model, add an `instructor` role to the view-as system, and build the tabbed admin UI at `/classes/admin/` so admins can build categories, instructors, classes, sessions, registrations, and discount codes end-to-end in the database. No public pages, no Stripe flow, no emails — those come in Plan 2. No instructor dashboard or calendar integration — those come in Plan 3.

**Architecture:** A new `classes/` Django app contains all models, managers, forms, admin views, and BDD specs. The admin UI lives in the hub's base template layout (not Django admin) with a tabbed sub-router at `/classes/admin/`. The `hub/view_as.py` role set gains a parallel `instructor` role derived from the existence of an `Instructor` record linked to `request.user`.

**Tech Stack:** Django 5.x, allauth (for instructor invite), PostgreSQL (production) / SQLite (local), Django ImageField (Pillow already present), HTMX + Alpine.js for admin tabs, pytest-describe + factory-boy for BDD specs.

**Spec reference:** `docs/superpowers/specs/2026-04-20-classes-design.md` — read sections 4, 5, 14 before starting.

## Design Summary (read before starting)

**Role model:** `hub/view_as.py` has roles `{member, guild_officer, admin}` today. Plan 1 adds a parallel `instructor` role, computed from whether `user.instructor` exists. The role is NOT in the hierarchy — it's capability-based. A user can have any combination.

**Instructor auth:** No `login_token`. Admin "invite instructor" creates a `User`, attaches a verified `allauth.account.EmailAddress`, creates an `Instructor` row, and sends a welcome email containing the hub login URL. The instructor logs in with email-code like any user.

**Model names:** `Class` is a Python keyword, so the model is `ClassOffering`. The UI label stays "Class."

**Member linking:** When an admin creates a Registration via the admin UI, the `save()` hook calls `link_member_by_email()` which matches against `allauth.account.EmailAddress` (verified only) and `MemberEmail`.

**Waiver defaults:** The `0002_seed_settings` data migration writes the global waiver and model-release text into `ClassSettings.objects.get_or_create(pk=1, defaults=...)`. Reverse is a no-op (`def reverse(apps, _): pass`).

**Admin visibility:** `/classes/admin/` is gated with `@user_passes_test(lambda u: u.is_authenticated and request.view_as.is_admin)` — but `request` isn't on the user, so use a decorator reading `request.view_as` directly. A new helper lives in `classes/views.py`: `admin_required`.

from datetime import date as date_type


class DiscountCode(models.Model):
    code = models.CharField(max_length=40, unique=True, help_text="Uppercase code — normalized on save.")
    description = models.CharField(max_length=255, blank=True, help_text="Admin-only description.")
    discount_pct = models.PositiveIntegerField(null=True, blank=True, help_text="Percent off (0-100).")
    discount_fixed_cents = models.PositiveIntegerField(null=True, blank=True, help_text="Flat cents off.")
    valid_from = models.DateField(null=True, blank=True, help_text="First date the code is valid.")
    valid_until = models.DateField(null=True, blank=True, help_text="Last date the code is valid.")
    max_uses = models.PositiveIntegerField(null=True, blank=True, help_text="Cap total uses. Null = unlimited.")
    use_count = models.PositiveIntegerField(default=0, help_text="Incremented on each successful registration.")
    is_active = models.BooleanField(default=True, help_text="Admin toggle to disable without deleting.")
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        ordering = ["code"]
        constraints = [
            models.CheckConstraint(
                check=(Q(discount_pct__isnull=False) | Q(discount_fixed_cents__isnull=False)),
                name="discount_has_value",
            ),
        ]

    def __str__(self) -> str:
        return self.code

    def save(self, *args, **kwargs) -> None:
        self.code = self.code.strip().upper()
        super().save(*args, **kwargs)

    def apply_to(self, price_cents: int) -> int:
        if self.discount_pct is not None:
            return int(price_cents * (100 - self.discount_pct) / 100)
        if self.discount_fixed_cents is not None:
            return max(0, price_cents - self.discount_fixed_cents)
        return price_cents

    def is_currently_valid(self) -> bool:
        if not self.is_active:
            return False
        today = date_type.today()
        if self.valid_from and today < self.valid_from:
            return False
        if self.valid_until and today > self.valid_until:
            return False
        if self.max_uses is not None and self.use_count >= self.max_uses:
            return False
        return True

from __future__ import annotations

from functools import wraps

from django.contrib.auth.decorators import login_required
from django.http import HttpResponseForbidden
from django.shortcuts import redirect, render


def admin_required(view_func):
    """Decorator: only admins (via request.view_as) may access."""
    @wraps(view_func)
    @login_required
    def wrapper(request, *args, **kwargs):
        view_as = getattr(request, "view_as", None)
        if view_as is None or not view_as.is_admin:
            return HttpResponseForbidden("Admin access required.")
        return view_func(request, *args, **kwargs)
    return wrapper

from django.contrib.auth import get_user_model
from django.utils.text import slugify


class InstructorInviteForm(forms.Form):
    display_name = forms.CharField(max_length=255)
    email = forms.EmailField()
    bio = forms.CharField(widget=forms.Textarea, required=False)

    def clean_email(self):
        email = self.cleaned_data["email"].lower().strip()
        User = get_user_model()
        if User.objects.filter(email__iexact=email).exists():
            raise forms.ValidationError("A user with this email already exists. Link them manually in Django admin.")
        return email

    def save(self):
        from allauth.account.models import EmailAddress
        from classes.models import Instructor

        User = get_user_model()
        email = self.cleaned_data["email"]
        user = User.objects.create_user(username=email, email=email)
        user.set_unusable_password()
        user.save()
        EmailAddress.objects.create(user=user, email=email, verified=True, primary=True)

        base_slug = slugify(self.cleaned_data["display_name"]) or "instructor"
        slug = base_slug
        n = 1
        while Instructor.objects.filter(slug=slug).exists():
            n += 1
            slug = f"{base_slug}-{n}"

        return Instructor.objects.create(
            user=user,
            display_name=self.cleaned_data["display_name"],
            slug=slug,
            bio=self.cleaned_data.get("bio", ""),
        )
